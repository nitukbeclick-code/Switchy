# חוסך · Switch AI (Switchy)

Israeli telecom price-comparison and **lead-generation** product — compare
cellular, internet, TV, triple bundles and abroad plans, see your potential
saving, and switch providers with step-by-step porting guidance.
**Hebrew-first, RTL.**

> **Honesty / E-E-A-T is a hard rule.** Every surface renders only real
> catalogue data and truthful copy — never fabricated plans, prices, ratings,
> reviews, or compliance claims.

## What's in the repo — four surfaces

| Surface | Path | Stack | Status |
|---------|------|-------|--------|
| Mobile/web app | `lib/` | Flutter (hand-authored, **no FlutterFlow**) | mobile + web target |
| Next.js GEO web app | `web/` | Next.js 16 (App Router), React 19, Tailwind v4, TS | **live** — `https://switchyy-omega.vercel.app` |
| Static marketing site | `site/` | Static HTML generated by `site/build.js` (Node) | **live** — `switchy-ai.com` |
| Backend + bots | `supabase/` | Postgres + RLS + `pg_cron` + Deno Edge Functions | live project `orzitfqmlvopujsoyigr` |

All four share one catalogue of real plans (owned by the Flutter app in
`lib/data/`, exported to the web surfaces) and one Supabase backend. A contact
request from **any** front-end converges on `public.leads`, which fans out to the
team's Telegram "digital rep" via the `notify-lead` edge function.

See **[`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md)** for the full picture and
data flow.

## Brand

White-glass + black-ink editorial base, green-forward (matching the Switchy
robot logo), with a disciplined two-accent system:

- **Green `#16A34A`** = ACTION (primary CTAs, active nav, links, focus).
- **Amber `#F59E0B`** = VALUE (savings figures, "best value" / win states).

Fonts: **Rubik** for display/headings, **Assistant** for body/labels. The
authoritative token rules (ink/glass palette, frosted-glass helpers, the
indigo→green alignment note) live in `CLAUDE.md` and
`lib/theme/app_theme.dart`.

## Quickstart

### Flutter app (`lib/`)

```bash
export PATH="$HOME/.flutter-sdk/bin:$PATH"
flutter pub get
flutter run                                          # device/emulator (LocalBackend)
flutter run --dart-define-from-file=dart_define.json # against Supabase (URL + anon key)
```

Validation gates (run in order, after every change):

1. `flutter analyze` → must print **No issues found**.
2. `flutter test` → all pass (the count only goes up).
3. `flutter build web --no-pub` → must reach **✓ Built build/web** (the critical
   web gate — catches `dart:io`/native code that breaks the web target).

`dart_define.json` (gitignored) supplies the live Supabase URL + **anon public**
key; without it the app runs fully on-device via `LocalBackend`. Never put the
`service_role` key in the client.

### Next.js web app (`web/`)

```bash
cd web
npm install
npm run dev     # next dev → http://localhost:3000
npm run build   # next build — the release gate; keep it GREEN
npm run lint    # eslint
npm run test    # vitest run
```

> ⚠️ Next 16 has breaking changes vs. older Next.js. Read
> `web/node_modules/next/dist/docs/` before writing Next-specific code
> (`web/AGENTS.md`). The build needs **no secrets** — the catalogue is bundled
> (`web/data/catalogue.json`). Runtime lead capture needs the server-only
> `SUPABASE_SERVICE_ROLE_KEY` (see `web/.env.local.example`).

### Static site (`site/`)

```bash
node site/build.js     # regenerate ~230 *.html + sitemap.xml + robots.txt; keep it GREEN
# or: npm run build:site
```

Reads the catalogue from `site/data/plans.json` and guide content from
`site/content/guides/*.json` (~142 guides). `index.html` is hand-written, not
generated. Deploy = publish `site/`.

### Supabase backend (`supabase/`)

Edge functions deploy via the Supabase CLI / GitHub Actions (separate from the
web deploys):

```bash
npx --no-install supabase functions deploy <slug> \
  --project-ref orzitfqmlvopujsoyigr --no-verify-jwt
deno task check   # type-check the functions
deno task test    # unit-test the functions
```

Schema/migrations are applied in the Supabase SQL editor (see
`supabase/README.md`). Full per-function detail in
**[`docs/EDGE_FUNCTIONS.md`](docs/EDGE_FUNCTIONS.md)**; the data model + RLS in
**[`docs/DATA_MODEL.md`](docs/DATA_MODEL.md)**.

## CI

`.github/workflows/ci.yml` runs the Flutter analyze/test/web-build/APK gates and
the Deno function type-check/tests; `deploy-functions.yml` ships edge functions
(manual or on a `[deploy]` commit); `bot-health.yml` probes the deployed
functions daily.
