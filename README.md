# חוסך (Chosech)

Israeli telecom price-comparison product — compare cellular, internet, TV,
triple bundles and abroad plans, see your potential saving, and switch
providers with step-by-step porting guidance. **Hebrew-first, RTL.**

The repo holds three surfaces:

| Surface | Path | Stack |
|---------|------|-------|
| Mobile/web app | `lib/` | Flutter (hand-authored, no FlutterFlow) |
| Marketing site | `site/` | Static HTML generated by `site/build.js`, deployed on Netlify/Vercel |
| Backend + bot | `supabase/` | Postgres schema + RLS + two Deno Edge Functions (the team's Telegram "digital rep") |

## The Flutter app (`lib/`)

- **Routing** — `go_router` in `lib/router.dart`. A `ShellRoute` hosts the
  bottom nav; full-screen routes (onboarding, auth, provider, search, savings,
  renewal-report…) sit outside it. Navigate with `context.goNamed/pushNamed`
  and `context.safePop()` (`lib/core/nav.dart`).
- **State** — `lib/app_state.dart`: one `ChangeNotifier` singleton over
  `SharedPreferences`, exposed via `Provider`. Persistence is incremental —
  each setter marks only the keys it changed; the four base64-bearing
  collections (community posts/replies, chat, advisor history) are written
  only by their own setters.
- **Design tokens** — `lib/theme/app_theme.dart`: `AppTheme.of(context)` for
  colours + text styles, `AppColors.*` for consts. Brand: primary `#15603E`
  (green), secondary `#C9EC4B` (lime), background `#F4F0E8`. Rubik for
  display/headings, Assistant for body, via `google_fonts`.
- **Services (the source of truth, pure & unit-tested — `lib/services/`)** —
  domain logic lives here so screens render its output instead of re-deriving
  it:
  - `recommendation_engine.dart` — explainable plan scoring over a
    `MatchProfile` (`MatchProfile.fromAppState(appState, cat)` is the canonical
    builder).
  - `advisor_engine.dart` — the AI-advisor's pure Hebrew NLU + recommendation.
  - `switch_economics.dart` — switching saving / break-even / milestone math.
  - `notifications.dart`, `renewal_report.dart`, `provider_ratings.dart`,
    `savings_summary.dart`, `search.dart`, `reminder_schedule.dart`,
    `media_service.dart` (+ web-safe `media_native_*.dart`).
  - `services/backend/` — a `Backend` seam: `LocalBackend` (default, no keys)
    or `SupabaseBackend` (live when build-time Supabase defines are present).
- **Shared widgets/components** — `lib/widgets/` (`AppButton`, `AppCard`,
  `SectionHeader`, `InfoBanner`, `EmptyState`, `StatPill`) and
  `lib/components/` (`PlanCardWidget`, `MiniPlanCard`, `LogoWidget`).
- **Pages** — one file per page under `lib/pages/<name>/<name>_widget.dart`.
- **Data** — `lib/data.dart` + `lib/data/plans_*.dart` (catalogue),
  `lib/models.dart`. Plan price suffixes come from `priceUnitLabel(plan)` /
  `priceUnitShort(plan)`; abroad plans carry a `priceUnit`
  (package/day/month/minute).

### Run & validate

```bash
export PATH="$HOME/.flutter-sdk/bin:$PATH"
flutter pub get
flutter run                       # device/emulator
flutter analyze                   # must print "No issues found"
flutter test                      # full suite
flutter build web --no-pub        # the critical web gate
flutter build apk / flutter build ipa
```

`dart_define.json` (gitignored) supplies the live Supabase URL/key; without it
the app runs on `LocalBackend`.

## Marketing site (`site/`)

Static, hand-tuned HTML. `node site/build.js` regenerates the category, guide,
provider, plans, compare and 404 pages from the templates plus the app
catalogue exported to `site/data/plans.json` (via
`flutter test tool/export_plans.dart`). `styles.css`/`script.js` are
content-fingerprinted (`?v=<hash>`) so cache-busting works under the immutable
headers in `netlify.toml` / `vercel.json`. Deploy = publish `site/`.

## Backend + Telegram bot (`supabase/`)

`schema.sql` defines the tables, RLS policies, anti-abuse lead gate and the
`pg_cron` schedules. Two Edge Functions (`functions/`, shared code in
`functions/_shared/`, unit-tested with `deno task test`):

- **`notify-lead`** — every new lead pings the team's Telegram chat with status
  buttons (claim / contacted / won / lost), an AI triage line, a prefilled
  WhatsApp opener, and chat commands (`/leads`, `/search`, `/stats`, `/hot`,
  `/weekly`).
- **`renewal-reminders`** — scheduled digest / delivery sweep / SLA follow-ups
  / weekly business report.

Setup and the secret-rotation runbook live in `supabase/README.md`;
`supabase/upgrade-2026-06-10.sql` is the single-paste delta for the live
project. Deploy is manual and separate from the app.

## CI

`.github/workflows/ci.yml` runs analyze + test + APK + web build on Flutter
3.38.10 and type-checks/tests the Deno functions; `bot-health.yml` probes the
deployed functions daily.
