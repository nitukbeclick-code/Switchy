# איסוף קטלוג מלא — מדריך ל-Claude in Chrome

המטרה: למלא לכל חבילה את **כל** הנתונים (כולל מחיר-אחרי-תקופה, נתב/ממיר,
דמי התקנה/חיבור, מגדיל טווח, כמות ערוצים) — ולהחזיר JSON **בדיוק בסכמה של
`site/data/plans.json`** (הסכמה למטה). **אסור להמציא נתון** — פרט שלא מופיע
באתר → `"לא מצוין"`.

> **לאן הנתונים באמת נכנסים (חשוב — צינור הקטלוג):** `site/data/plans.json`
> הוא **קובץ מיוצא**, לא מקור הכתיבה. מקור הכתיבה הוא ה-Dart הידני
> (`lib/data/plans_*.dart`), ש-`flutter test tool/export_plans.dart` מייצא
> ממנו את `plans.json` ומזריע (insert-only) מזהים חדשים ל-`public.plans`.
> בזמן ריצה **ה-DB הוא הסמכותי**: האתר נבנה קודם מקריאה חיה של `public.plans`
> ונופל ל-`plans.json` רק כשהקריאה נכשלת. לכן:
>
> - **חבילה חדשה** → מוסיפים אותה ל-`lib/data/plans_*.dart` ומריצים את
>   הייצוא (עריכה ידנית של `plans.json` תימחק בייצוא הבא ולא תגיע לאפליקציה).
> - **עדכון מחיר/פרט לחבילה קיימת** → עורכים ישירות את השורה ב-`public.plans`
>   בדשבורד Supabase (הייצוא לעולם לא דורס שורה קיימת, אז העריכה שורדת).

> למה Chrome ולא סוכן אוטומטי: אתרי התקשורת מחזירים 403 לגישה אוטומטית,
> והנתונים נפתחים רק בלחיצה על כפתור "מידע נוסף" (JavaScript). רק דפדפן
> אמיתי (Claude in Chrome) יכול ללחוץ ולקרוא אותם.

## איך משתמשים

1. פותחים את אתר החברה (כתובת + מיקום הכפתורים בטבלה למטה).
2. מדביקים ל-Claude in Chrome את **הפרומפט** למטה + הבלוק של אותה חברה.
3. Claude לוחץ על כל כפתור "מידע נוסף", קורא, ומחזיר מערך JSON.
4. מתרגמים את הבלוקים לערכי `Plan` ב-`lib/data/plans_*.dart` (חבילות חדשות)
   או מעדכנים שורות קיימות ב-`public.plans` בדשבורד; אחר כך
   `flutter test tool/export_plans.dart` מרענן את `plans.json`,
   ו-`node site/build.js` בונה את האתר.

### פרומפט מוכן להדבקה (לכל חברה)

```
עבור על כל החבילות בעמוד הזה. לכל חבילה לחץ על כפתור המידע/"לפרטים נוספים"/
"מידע נוסף" כדי לחשוף את כל הנתונים (סגור מודאל לפני פתיחת הבא). החזר מערך JSON
בלבד, אובייקט אחד לכל חבילה, בדיוק לפי הסכמה שאתן לך. אם פרט לא מופיע באתר —
כתוב "לא מצוין". אל תמציא מחירים. חשוב במיוחד: מחיר המבצע, המחיר אחרי סיום
התקופה (מופיע לרוב בדיסקליימר התחתון), דמי התקנה/חיבור, מחיר נתב/ממיר, ומגדיל טווח.
```

## סכמת ה-JSON (שדה-שדה)

הקובץ עצמו הוא `{ "generated": <ISO>, "categories": [...], "plans": [...] }`;
כל חבילה במערך `plans` נושאת את **כל** השדות הבאים (גם כשהערך `null`/ריק):

```jsonc
{
  "id": "net_bezeq_bfiber300",   // מזהה יציב: <תחילית>_<ספק>_<שם-מקוצר-באנגלית>
  "cat": "internet",             // cellular | internet | tv | triple | abroad (+ electricity בקטלוג המלא)
  "provider": "בזק",             // שם ספק קנוני בדיוק (ראו רשימה למטה — חשוב ללוגו)
  "plan": "bFiber 300 מגה",      // שם החבילה כפי שמופיע
  "price": 109,                  // מחיר מבצע (מספר, ₪), מעוגל. היחידה לפי priceUnit
  "priceExact": null,            // מחיר עם אגורות (69.9) אם לא עגול; אחרת null
  "after": 196,                  // מחיר אחרי סיום התקופה (מעוגל). null אם אין/קבוע לתמיד
  "afterExact": null,            // "אחרי" עם אגורות אם לא עגול; אחרת null
  "net": "סיב אופטי",            // 5G | 4G | סיב אופטי | כבלים | סטרימינג | בינלאומי | eSIM
  "is5G": false,                 // boolean
  "noCommit": true,              // boolean — ללא התחייבות
  "hasAbroad": false,            // boolean — כולל גלישה בחו״ל
  "priceUnit": "month",          // month | package | day | minute (כמעט תמיד month)
  "specs": {                     // המספרים התיאוריים (לא עלויות) — לפי קטגוריה (ראו למטה)
    "מהירות": "עד 300/100"
  },
  "feats": ["סיב אופטי", "נתב כלול לחודשיים"],  // בולטים קצרים
  "kind": "regular",             // regular | dataonly (גלישה בלבד) | kosher (מסלול כשר)
  "fees": {                      // עלויות חד-פעמיות/ציוד — אובייקט מפתח→ערך (טקסט חופשי).
    "התקנה": "₪49",              //   מפתחות בשימוש בקטלוג: "התקנה", "דמי חיבור", "נתב",
    "נתב": "+₪19.9/ח׳"           //   "מגדיל טווח", "ממיר", "SIM", "eSIM", "הצטרפות".
  },                             //   ערכים כמו "חינם" / "אין" / "לא מצוין" תקפים.
  "fineLines": [                 // ה"אותיות הקטנות" — מדרגות מחיר, חריגות, תנאי מבצע
    "מחיר כולל נתב לחודשיים בלבד"
  ],
  "terms": [],                   // בולטי התחייבות/חוזה (לרוב ריק)
  "term": null,                  // טקסט תקופת התחייבות, או null
  "eligibility": null,           // תנאי זכאות ("לבעלי תשתית בזק פעילה"), או null
  "notes": null,                 // הערה חופשית, או null
  "updatedAt": "2026-06"         // חודש העדכון (YYYY-MM)
}
```

> **אין שדות שטוחים `setupFee` / `equipment` / `rangeExtender`** — עלויות
> התקנה/ציוד/מגדיל-טווח נכנסות כמפתחות בתוך `fees{}`, ופרטי דיסקליימר
> (מדרגות מחיר, "אחרי שנה…") נכנסים ל-`fineLines[]`. נתון שנאסף בשדה שטוח
> לא ייקלט.

### שמות ספקים קנוניים (חובה בדיוק — אחרת הלוגו לא יוצג)
`בזק` · `סלקום` · `פרטנר` · `פלאפון` · `גולן טלקום` · `HOT` · `הוט מובייל` ·
`Xphone` · `רמי לוי` · `WeCom` · `NextTV` · `yes` · `גילת` · `וואלה מובייל` ·
`019 מובייל` · `CCC` · `STING TV` · `Airalo eSIM`

### תחיליות id לפי קטגוריה
`cellular → cel_` · `internet → net_` · `tv → tv_` · `triple → tri_` · `abroad → ab_` · `electricity → el_`
(חשמל נאסף בנפרד — לא חלק ממדריך ה-Chrome הזה, אבל הקטגוריה נתמכת בקטלוג וב-build.)

### מפתחות specs לפי קטגוריה
| קטגוריה | מפתחות specs מומלצים |
|---------|----------------------|
| cellular | `נתונים` (GB), `דקות`, `SMS`, `חו"ל` |
| internet | `מהירות` (למשל "עד 1000/100") |
| tv | `ערוצים` (למשל "~140"), `ממירים`, `VOD` |
| triple | `ערוצים`, `מהירות`, `סטרימרים`, `ממירים` |
| abroad | (לרוב ריק; הפרטים ב-feats) |

> שים לב: עלויות ציוד/התקנה/מגדיל-טווח הולכות ל-`fees{}` (מפתחות "התקנה" /
> "נתב" / "מגדיל טווח" / "ממיר" / "דמי חיבור"), **לא** ל-specs. specs = מספרים
> תיאוריים בלבד.

## בדיקת תקינות אחרי עדכון
1. `node -e "JSON.parse(require('fs').readFileSync('site/data/plans.json','utf8'))"` — תקינות JSON.
2. `node site/build.js` — חייב להסתיים ב-"Generated …" בלי שגיאות. שימו לב
   לשורת `Catalogue source:` בתחילת הפלט — כשהיא `LIVE Supabase`, האתר נבנה
   מ-`public.plans` החי ו-`plans.json` המקומי משמש רק כגיבוי (כדי שחבילה
   חדשה תופיע, היא צריכה להגיע ל-DB דרך הייצוא, או שהקריאה החיה תיכשל).
3. לפתוח `plans.html` / `compare.html` ולוודא שהשדות החדשים מופיעים.
